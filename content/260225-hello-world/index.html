Lets start with hosting static content on github pages: 
   https://docs.github.com/en/pages/quickstart
Follow instructions, key point which escaped me initially, is that naming repo in specific way "[username].github.io" publishes repo as root of domain (so we have "[username].github.io/index.html" instead of "[username].github.io/[reponame]/index.html"). Now, I do not want to use Jekyll (would rather generate my own static pages), and I need to publish only a specific direcory as root, so I need stock "Deploy static content to Pages" but with additional config under "Checkout" step and "Upload artifact" step that point to specific directory. 
   https://github.com/mstandio/mstandio.github.io/blob/main/.github/workflows/static.yml
